# 问卷系统 - 系统说明

## 1. 项目概述

本项目是一个基于 **MongoDB** 的在线问卷系统，目标是实现一个简化版的问卷平台，支持用户注册登录、创建问卷、发布问卷、填写问卷以及查看问卷统计结果等核心功能。

系统重点围绕以下几个方面展开：

- 使用 MongoDB 完成问卷系统的数据建模（充分利用文档型数据库的优势）。
- 使用后端程序（FastAPI）实现问卷创建、答卷提交、统计分析等业务逻辑。
- 支持各种题型的校验规则与问卷的跳转逻辑处理（数据驱动，完全不硬编码）。
- 实现跳转逻辑的完整性验证（目标存在性、向前跳转禁止、循环检测）。
- 全开发流程使用 AI 工具辅助系统设计、代码生成与文档整理。

本系统已完成**第一、二阶段的全部需求**。第一阶段建立了问卷核心功能框架（创建、发布、填写、统计），第二阶段进一步扩展了题目能力，引入了**版本管理、共享协作、题库、跨问卷统计**等企业级特性，使问卷系统成为一个灵活的问卷内容管理平台。

## 3. 技术选型说明

### 3.1 后端：Python + FastAPI

后端采用 Python + FastAPI 进行开发，主要优势在于：

- **开发效率高**：语法简洁，且得益于 Pydantic，能快速完成复杂嵌套 JSON 数据的解析与验证（如问卷题目和跳转逻辑）。
- **自带文档**：自动生成 Swagger/OpenAPI 文档，极大地方便了开发过程中的接口联调。
- **现代化**：原生支持类型标注，利于规范代码和防错。

### 3.2 数据库：MongoDB + PyMongo

数据库采用 MongoDB，驱动选择 PyMongo：

- **Schema 灵活**：第二阶段将题目从问卷内嵌抽取为独立集合，支持题目版本历史链（通过 `versions` 数组和 `parent_version_number`），答案内嵌答卷记录，能有效减少关联查询，充分发挥 MongoDB 处理树状 JSON 结构和版本控制的优势。
- **易于拓展**：对于多变的问卷校验规则（如不同题型对应的校验字段不同）和版本分支结构，Schema-less 特性避免了关系型数据库复杂的表结构修改。
- **并发安全**：版本号递增通过 MongoDB 的原子操作 `$inc` 保证，版本链永不覆盖（仅 `$push` 追加）。

### 3.3 前端：React + Vite + TypeScript

- **组件化交互**：React 极其适合实现动态表单、题目编辑器、问卷填写控制等交互密集的页面。
- **类型安全**：TypeScript 能提升代码可维护性，减少前后端字段对接时的低级错误。

### 3.4 认证：JWT（PyJWT）

系统采用 JWT 保持无状态认证：

- 生成 token 后在前端缓存，所有需要权限获取或提交的请求均携带 `Bearer Token`，实现前后端分离鉴权。

## 4. 系统功能概述

### 4.1 用户账号功能

- 面向所有人的注册功能（提供唯一的用户名、密码）。
- 用户登录获取鉴权凭证（Token）。
- 独立的工作台，用户登录后仅查看及管理自己创建的问卷。

### 4.2 问卷创建与管理

- 问卷元数据管理（标题、描述、匿名设置、是否允许重复填写的限制、截止时间）。
- 问卷状态生命周期管理（创建草稿 -> 发布收集 -> 关闭停止收集）：
  - 草稿状态：完全开放编辑（题目结构、元信息）
  - 发布状态：禁止所有编辑（确保数据收集稳定）
  - 关闭状态：限制编辑（可修改元信息，可升级题目版本号）
- 题目引用模式（第二阶段）：
  - 问卷中的题目改为引用模式 `(question_ref_id, version_number)` 而非内嵌
  - 编辑时可切换题目的不同版本（若题目有多个版本可选）
  - 已发布/已关闭问卷锁定题目版本不可更改
- 通过访问码（Access Code）生成对外的安全分享链接（如 `/survey/xxxxxx` 形式）。
- 匿名模式：当创建者开启允许"匿名填写"时，虽然需要登录拦截，但在数据库存储和统计时会隐去答题者的真实身份。

### 4.3 题目库与管理（第二阶段新增）

- **独立题目集合**：题目不再内嵌于问卷，而是作为独立的、可复用的实体存储在 `questions` 集合中。
- **题目版本控制**：每个题目维护完整的版本历史链（v1、v2、v3...）：
  - 创建题目时自动生成 v1
  - 修改题目时创建新版本，原版本永不删除
  - 版本链支持分支结构（通过 `parent_version_number` 追溯血缘）
  - 已发布/已关闭问卷的版本冻结保护（不能被原地修改）
- **题目共享**：创建者可以将题目共享给其他用户：
  - 被共享者可查看、引用、创建新版本
  - 被共享者不能删除或取消共享（仅创建者可）
  - 被共享者不能原地修改版本（只能创建新版本）
- **题库管理**：用户可以将题目（自创或他人共享的）加入自己的题库：
  - 题库作为个人的题目收藏夹，方便后续快速选题
  - 从题库移出不会删除题目本身
  - 支持跨问卷、跨用户的题目复用
- **引用反查**：系统追踪每个题目版本被哪些问卷使用，防止已发布问卷的版本被删除。

### 4.4 跳转逻辑与条件控制

- 设定题目间的跳转逻辑（根据不同选项或填写的数字，条件分支跳转）：
  - 支持三种条件类型：单选匹配、多选包含（any/all）、数字比较（7种操作符）
  - 支持两种动作：跳转到指定题目、结束问卷
  - 后端验证：跳转目标存在性、禁止向前跳转、循环跳转检测
  - 前端限制：编辑器中跳转目标下拉框只显示后续题目
- 丰富的题型支持（单选题、多选题、文本填空题、数字填空题）与精细化校验规则（如多选的数量限制、填空的字数、数字的最值及整数校验）。
- 逻辑执行：填写时前端会实时校验合法性，引擎会根据用户的选择自动执行跳题逻辑。

### 4.5 统计分析功能

- 按问卷统计：
  - 宏观统计：问卷整体的答卷收集数量概览
  - 微观统计：
    - 单选题/多选题：精准统计每个选项的被选次数及占比
    - 文本题：回收所有的内容列表
    - 数字题：不仅回收内容列表，还将自动计算填写数值的平均值等统计指标
- **跨问卷统计**（第二阶段新增）：
  - 选择单个题目，统计其在多个问卷中的综合数据
  - 按题目版本分组统计（因为不同版本的选项可能不同）
  - 显示哪些问卷中使用了该题目及其版本
  - 支持版本演变过程中的统计对比
