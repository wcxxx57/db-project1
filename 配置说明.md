# 项目配置说明

本项目为基于 MongoDB 的在线问卷系统，包含前后端两个子项目。

为方便运行和测试，系统已默认配置好了一套位于阿里云的高可用 MongoDB 云数据库，您可以直接启动前后端进行测试，**无需在本地额外安装 MongoDB 服务**。

---

## 1. 环境准备

运行本项目前，请确保您的计算机已经安装以下基础环境：

- **Node.js** (推荐 >=18.0) 及 npm（用于启动前端应用）
- **Python** (推荐 >=3.10) 及 pip（用于启动后端应用）

---

## 2. 数据库配置（可选）

为方便快速上手，代码中已经硬编码（默认加载）了一个可以一直保持访问的公网连通的 MongoDB 实例。因此您不需要启动本地 MongoDB 服务也能让项目正常工作。

**（对于高级使用者：如果您希望连接到您自己的本地或其他环境的 MongoDB 实例）**
您可以在 `backend` 目录下新建一个名为 `.env` 的环境变量文件，写入如下内容直接覆盖默认配置：

```env
MONGODB_URI=mongodb://localhost:27017/admin
MONGODB_DB_NAME=your_db_name
JWT_SECRET_KEY=your_custom_secret_key
```

---

## 3. 启动后端 (Backend)

后端采用 Python + FastAPI 框架构建。

1. **进入后端目录**：

   ```bash
   cd backend
   ```

2. **创建并激活虚拟环境（推荐）**：

   ```bash
   # Windows 系统：
   python -m venv .venv
   .\.venv\Scripts\activate

   # 补充：macOS / Linux 系统请执行 `source .venv/bin/activate`
   ```

3. **安装依赖**：

   ```bash
   pip install -r requirements.txt
   ```

4. **启动服务**：
   ```bash
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```
   > 启动成功后，后端 API 服务将运行在：`http://localhost:8000`
   > 同时 FastAPI 自动生成的交互式 API 调试文档（Swagger UI）可访问：`http://localhost:8000/docs`

---

## 4. 启动前端 (Frontend)

前端采用 React + Vite + TypeScript 构建。

1. **打开一个新的终端命令窗口，并进入前端目录**：

   ```bash
   cd frontend
   ```

2. **安装依赖包**：

   ```bash
   npm install
   ```

   > 注：如果网络较慢，可使用淘宝镜像加速 `npm install --registry=https://registry.npmmirror.com`

3. **启动开发服务器**：
   ```bash
   npm run dev
   ```

启动成功后，终端将输出前端运行访问地址（通常为：`http://localhost:5173/`）。
直接点击或复制该地址到浏览器中打开即可进入问卷系统。

---

## 5. 项目初始化流程建议

1. 打开浏览器进入系统前端（`http://localhost:5173/`）。
2. 在登录浮窗处，因为是第一次使用，可随意填写一套账号密码点击【注册】。
3. 注册成功后系统将自动保存状态。刷新或点击【创建新问题】按钮进入设计页面，体验全部流程。

## 6. 端口占用注意事项

- 请确保本地的 `8000` 端口不被占用，否则后端启动会报错。
- 前端使用 Vite，若 `5173` 被占用它会自动顺延端口（如 `5174` 等），直接以终端输出打印的访问地址为准即可。
